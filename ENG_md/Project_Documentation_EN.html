Interactive Stock Theory Lab

Hackonomics 2026 Final Project Documentation (English)

Version: Final / Date: 2026-03-30

1. Project Overview

Interactive Stock Theory Lab is an educational web application that helps users learn finance theories with overseas stock market data. Its core objective is not short-term prediction but demonstrating that the same market data can lead to different conclusions depending on model assumptions.

2. Motivation and Objectives

  • Reduce learning barriers when finance is studied only through formulas
  • Connect real market behavior to theoretical frameworks
  • Build an educational lab for assumptions, sensitivity, and model interpretation rather than investment recommendation

3. Core Value Proposition

  • Non-predictive design: A theory application tool, not a prediction engine
  • Comparative learning: Multiple theories compared on one screen
  • Interactive analysis: What-if controls update outcomes instantly
  • Explainability: Structured "why this happened" interpretation per theory

4. User Flow

  1. User enters a ticker (e.g., AAPL, MSFT, NVDA, SPY)
  2. User selects period (1M/6M/1Y/3Y/5Y/Custom) and theories
  3. User runs analysis and data is fetched/normalized/computed
  4. User reviews comparison table and theory cards
  5. User changes assumptions via sliders and observes differences

6. Technical Architecture

6.1 Frontend

  • HTML + CSS + JavaScript
  • Tailwind CSS (CDN), Chart.js
  • Card-based responsive UI

6.2 Backend

  • Node.js built-in HTTP server
  • Proxy endpoint: /api/price
  • Alpha Vantage integration, in-memory cache, request queue throttling

7. Data and Computation Pipeline

  1. Receive ticker/range parameters
  2. Call external API (daily/weekly auto-selection)
  3. Normalize to {t,o,h,l,c,v}
  4. Compute returns, regression, strategy, and risk metrics
  5. Render comparison table and charts
  6. Recompute instantly on what-if control changes

9. Local Run Guide

  1. Confirm Node.js installation (node -v)
  2. Set API key in .env
  3. Run node server.js
  4. Open http://localhost:3000 in browser

10. Output and Interpretation

  • At least three KPIs per selected theory
  • Charts + tables + explanation text shown together
  • Interpretation pattern: assumption -> observed feature -> output meaning
  • Sensitivity controls to show assumption-driven conclusion changes

11. Challenges and Resolutions

  • API limits: free-endpoint fallback, throttling, and cache strategy
  • CORS/key exposure: server-side proxy architecture
  • Responsive chart overflow: chart container and axis/tick tuning
  • Explanation quality: improved beginner-friendly “why” narratives

14. Conclusion

This project increases practical understanding of finance theories by mapping them to real market behavior. It is intentionally designed as an educational interpretation platform, not an investment decision engine.
